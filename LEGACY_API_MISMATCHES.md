# Bollard Legacy Adversarial Test File API Mismatches

## Summary
The legacy adversarial test files were generated by an older version of the adversarial tester (likely before Stage 1.5 completion). They use outdated type shapes and API signatures that have evolved significantly. This document maps all the required changes to modernize them.

---

## 1. BLUEPRINT TYPES (packages/engine/src/blueprint.ts)

### Current API
```typescript
interface BlueprintNode {
  id: string
  name: string
  type: NodeType  // "deterministic" | "agentic" | "risk_gate" | "human_gate"
  execute?: (ctx: PipelineContext) => Promise<NodeResult>
  agent?: string
  postconditions?: ((ctx: PipelineContext) => boolean)[]
  onFailure?: "stop" | "retry" | "skip" | "hand_to_human"
  maxRetries?: number
}

interface NodeResult {
  status: "ok" | "fail" | "block"
  data?: unknown
  cost_usd?: number
  duration_ms?: number
  error?: NodeResultError
  probes?: ProbeDefinition[]
}
```

### Legacy Test Misconceptions (from runner.adversarial.test.ts)
| Legacy | Current | Files | Issues |
|--------|---------|-------|--------|
| `description` property on Blueprint | No such field | runner.adversarial.test.ts:20, 66 | Blueprints have `id` and `name`, not `description` |
| `agent` object with `role`, `goal`, `backstory` | `agent` is just a string (agent ID) | runner.adversarial.test.ts:26-30, 73-75, etc. | Agent definitions are separate; node just refs them by string |
| `tools` array on BlueprintNode | Nodes don't have tools; AgentDefinition does | runner.adversarial.test.ts:31, 74, etc. | Legacy confuses node structure with agent structure |
| `dependencies` array on BlueprintNode | No such field | runner.adversarial.test.ts:32, 83, etc. | No explicit dependency tracking in node; sequential order in nodes array |
| `message` field on human_gate nodes | No such field | runner.adversarial.test.ts:161 | Not part of BlueprintNode |
| Blueprint with `maxCostUsd` and `maxDurationMinutes` | These fields exist but legacy tests don't set them | runner.adversarial.test.ts (all) | All test blueprints missing these required fields |
| NodeResult with `output` field | Should be `data` | runner.adversarial.test.ts:40, 89, 104, etc. | Legacy uses `output`; current uses `data` |
| NodeResult with `costUsd` (snake_case) | Field is `cost_usd` | runner.adversarial.test.ts:41, 92, etc. | Naming changed to snake_case |
| NodeResult with `durationMs` (camelCase) | Field is `duration_ms` | runner.adversarial.test.ts:42, 93, etc. | Naming changed to snake_case |
| runId UUID format assumption | runId is `YYYYMMDD-HHMM-prefix-slug-hex` | runner.adversarial.test.ts:59 | Not 36-char UUID format; check actual implementation |

## CHANGE SUMMARY FOR CURSOR PROMPT

### Find & Replace Patterns (High Priority)

1. **BlueprintNode fields:**
   - Add `maxCostUsd` and `maxDurationMinutes` to all Blueprint objects
   - Replace all `{ id, name, type, agent: {...}, tools: [], dependencies: [] }` with proper `{ id, name, type, agent: "string", ...}` format
   - Remove `description` from Blueprint objects
   - Remove `message` from human_gate nodes
   - Remove `tools`, `dependencies`, `backstory`, `goal` from node agent definitions

2. **NodeResult fields:**
   - Replace `output` → `data`
   - Replace `costUsd` → `cost_usd`
   - Replace `durationMs` → `duration_ms`

3. **AgentDefinition fields:**
   - Replace `name` → `role`
   - Replace `instructions` → `systemPrompt`

4. **AgentContext structure:**
   - Replace `{ messages: [], variables: new Map() }` → `{ pipelineCtx, workDir, allowedCommands?, progress? }`

5. **AgentResult fields:**
   - Replace `content` → `response`
   - Replace `usage` → use `totalCostUsd`, `totalDurationMs`, `turns`, `toolCalls`
   - Replace `messages` → (remove; not part of result)

6. **LLMResponse usage:**
   - Replace `prompt_tokens`, `completion_tokens`, `total_tokens` → `inputTokens`, `outputTokens`
   - Add `costUsd` to all mock responses
   - Change `content` from string to `LLMContentBlock[]`

7. **CostTracker method names:**
   - Replace `addCost()` → `add()`
   - Replace `getTotalCost()` → `total()`

8. **MockProvider:**
   - Constructors need proper `LLMResponse` objects with all fields
   - Responses must have `{ content: LLMContentBlock[], stopReason, usage, costUsd }`

9. **Function calls:**
   - Remove `await` from `createPlannerAgent()` (or add it if actually async)
   - Pass proper `AgentContext` to `executeAgent()`

10. **Remove entire test suites:**
    - `runner.adversarial.test.ts`: All "circular dependencies" and "missing dependencies" tests (lines 421-471)
    - These tests assume node-level dependency graph that doesn't exist

### Medium Priority

- Verify runId format expectations (not standard UUID)
- Verify ToolchainProfile structure matches current detect/types.ts
- Update agent creation calls to use proper profiles
- Validate config.ts test expectations against current API
- Remove all cross-module type imports that are internal

### Low Priority

- Clean up comments and descriptions
- Standardize assertion messages
- Consolidate similar test cases
- Add missing test coverage for new API features
